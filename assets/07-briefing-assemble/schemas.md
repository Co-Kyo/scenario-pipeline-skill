# briefing 产出格式

## 路径

`{workDir}/.meta/briefings/{seq}-{short_name}.md`

## Markdown 模板

```markdown
# 长列表渲染 — 组装 Briefing

## 命题信息
命题：长列表渲染：如何在万级数据量下保持流畅滚动
限定词：（无）

## 涉及能力摘要
### A1-浏览器渲染管线 [用于: overview/edge-cases/trade-offs/experiment]
机制：...
瓶颈：
  - [名称]：[触发条件] → [表现症状]
权衡：
  - [维度]：[方案A] vs [方案B]，建议 [选择建议]
实验代码：（如有）
参考：（URL 列表）

## 内容比例约束
开篇 10-15%：无限定词，直接切入场景
主体 70-80%：通用工程原理
收尾 10-15%：通用方案总结
```

## 字段说明

| 字段 | 含义 |
|------|------|
| `机制` | 能力的核心技术原理（mechanism_summary） |
| `瓶颈` | 与该命题相关的工程瓶颈 |
| `权衡` | 该能力涉及的典型权衡 |
| `参考` | 按 Tier 排序的参考资料 |
