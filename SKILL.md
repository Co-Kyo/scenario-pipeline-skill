# Scenario Pipeline

将前端技术文章转化为三层结构化知识产品：
- **命题研究** → overview + edge-cases + trade-offs + experiment
- **能力知识库** → capabilities/{id}-{name}.md（跨命题原子能力）
- **学习阶梯** → learning-ladder.md（渐进式引导路径）

## 调用方式

使用自然语言显式调用，推荐以下句式：

| 场景 | 推荐句式 |
|------|----------|
| 完整流程 | "使用 scenario-pipeline，对 <场景描述> 进行完整研究" |
| 仅前处理 | "使用 scenario-pipeline，对 <信息源> 进行前处理" |
| 仅后处理 | "使用 scenario-pipeline，从能力研究开始，处理 <场景>" |
| 断点续写 | "使用 scenario-pipeline，从 Step <N> 继续处理 <场景>" |

系统从自然语言自动推断：经验年限、研究深度、目标平台等约束。

## 流程总览

⚠️ 每步只读该步文件，严禁提前加载后续步骤（详见 rule-isolation.md）。

### 完整流程

```
初始化 → 意图锚定 → 头脑风暴 → 依赖分区 → 前处理 → 后处理
         (00)        (01)        (02)      (03-05)   (06-09)
```

### 步骤详情

| # | 步骤 | 核心目的 | 关键产出 |
|---|------|----------|----------|
| 00 | 意图锚定 | 解析用户指令，推断年限，生成共享骨架 | `anchors.json` |
| 01 | 头脑风暴 | 4维度Agent并行分析，产出结构化需求网 | `requirement-web.json` |
| 02 | 依赖分区 | 整理命题依赖DAG，识别分区点分批执行 | `partition-analysis.json` |
| 03 | 广域扫描 | 按level_weight差异化搜索信源，结构化提取 | `.raw-materials/` |
| 04 | 能力图谱 | 跨命题去重合并原子能力，计算战略价值 | `capability-graph.json` |
| 05 | 评估入池 | 四维评估矩阵打分，确定优先级和学习顺序 | `evaluations.json` |
| 06 | 能力研究 | 深度研究原子能力，产出知识库主文件 | `capabilities/*.md` |
| 07 | Briefing组装 | 从能力摘要提取关键信息，组装Briefing | `briefings/*.md` |
| 08 | 命题组装 | 组装四象限研究输出（overview/edge-cases/trade-offs/experiment） | `{seq}-{short_name}/` |
| 09 | 学习阶梯 | 生成从"不会"到"能讲"的渐进式路径 | `learning-ladder.md` |

### 阶段划分

| 阶段 | 步骤 | 说明 |
|------|------|------|
| **初始化** | - | 确认 workDir（交互步骤），各步骤按需加载公共规则 |
| **意图锚定** | 00 | 解析指令，推断年限，生成共享骨架 |
| **头脑风暴** | 01 | 条件触发：topic含抽象维度词时执行 |
| **依赖分区** | 02 | 命题依赖分析，分批执行规划 |
| **前处理** | 03→04→05 | 串行：扫描→能力图谱→评估入池 |
| **后处理** | 06→07→08→09 | 串行：能力研究→Briefing→组装→学习阶梯 |

### 初始化规则

执行任何步骤前，必须先完成初始化：

1. **确认 workDir**：向用户展示将要使用的输出目录，等待确认
   - 如果用户未指定，默认使用 `{当前日期}-{场景简称}` 作为 workDir
   - 用户可修正 workDir 路径
2. workDir 确认后，后续所有步骤的产出路径均基于此目录派生

## 执行

执行 Step N 时引用 Step N+1 文件内容即为违规。
