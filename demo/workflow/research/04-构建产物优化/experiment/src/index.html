💾 持久化缓存策略

vendor.[hash].jsCache-Control: max-age=31536000 (1年)
app.[hash].jsCache-Control: max-age=31536000 (1年)
chunk-*.jsCache-Control: max-age=31536000 (1年)
index.htmlCache-Control: no-cache (协商缓存)
// 核心原则：文件名带 contenthash + 长期缓存 // 内容不变 → hash 不变 → 缓存命中 // 内容变化 → hash 变化 → 获取新文件 // nginx 配置 location /assets/ { add_header Cache-Control "public, max-age=31536000, immutable"; } location / { add_header Cache-Control "no-cache"; // HTML 协商缓存 } // chunk 粒度控制 // 太粗 → 一个小改动整个 vendor 缓存失效 // 太细 → HTTP 请求数过多（HTTP/2 多路复用缓解） // 建议：框架 + 大型库 各一个 chunk，业务代码按路由分割
