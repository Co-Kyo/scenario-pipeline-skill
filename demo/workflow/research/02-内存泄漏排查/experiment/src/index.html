💥 泄漏 1: Detached DOM P0-必现

DOM 从页面移除，但 JS 仍持有引用 → GC 无法回收

// ❌ 泄漏代码 const nodes = []; function addCard() { const el = document.createElement('div'); el.textContent = 'I am leaked'; container.appendChild(el); container.removeChild(el); // 从 DOM 移除 nodes.push(el); // 但 JS 仍持有引用！GC 无法回收 } // ✅ 修复：移除引用 function removeCard() { container.removeChild(el); el = null; // 断开引用，允许 GC 回收 }

💥 泄漏 2: 事件监听未解绑 P0-必现

组件销毁后事件监听器仍在 → 回调持有闭包引用 → 内存增长

// ❌ 泄漏：组件销毁未移除监听 useEffect(() => { window.addEventListener('scroll', onScroll); // 缺少 return () => window.removeEventListener('scroll', onScroll); }, []); // ✅ 修复：cleanup 函数 useEffect(() => { window.addEventListener('scroll', onScroll); return () => window.removeEventListener('scroll', onScroll); }, []);

💥 泄漏 3: 全局缓存无界增长 P0-必现

Map/Set 不断添加但无淘汰策略 → 老生代持续上升

缓存条目:0
缓存类型:Map (强引用)
// ❌ Map 强引用，永不回收 const cache = new Map(); items.forEach(item => cache.set(item.id, item.data)); // ✅ WeakMap 允许 GC 回收 const cache = new WeakMap(); items.forEach(item => cache.set(item, item.data)); // 当 item 被 GC 回收，cache 中的条目自动消失

🔍 DevTools 检测方法 A4-DevTools

**Heap Snapshot 对比法（定位 Detached DOM）：** 1. Chrome DevTools → Memory → Take heap snapshot (Snapshot 1) 2. 执行可疑操作（如反复切换路由） 3. 再次 Take heap snapshot (Snapshot 2) 4. Comparison 视图 → 筛选 "Detached" 关键字 5. 看 #Delta 列：正数 = 新增的 Detached 节点 = 泄漏 **Allocation Timeline（实时追踪）：** 1. Chrome DevTools → Memory → Allocation instrumentation on timeline 2. 执行操作，观察蓝色竖线（分配）是否持续增长 3. 停止操作后，蓝色不回落 = 内存泄漏 **Performance Monitor（快速巡检）：** 1. Chrome DevTools → Performance Monitor（左侧面板） 2. 观察 JS heap size 趋势：锯齿形 = 正常 GC，持续上升 = 泄漏 3. 观察 DOM Nodes 趋势：操作后不回落 = Detached DOM 泄漏

// AbortController 统一取消模式 function loadData(url, signal) { return fetch(url, { signal }) .then(res => res.json()) .catch(err => { if (err.name === 'AbortError') return null; // 正常取消 throw err; }); } // 组件级别 useEffect(() => { const controller = new AbortController(); loadData('/api/data', controller.signal); return () => controller.abort(); // 统一清理 }, []); // WeakRef + FinalizationRegistry 模式 class Cache { #cache = new Map(); #registry = new FinalizationRegistry(key => { this.#cache.delete(key); // GC 回收时自动清理 }); set(key, value) { const ref = new WeakRef(value); this.#cache.set(key, ref); this.#registry.register(value, key); } get(key) { const ref = this.#cache.get(key); return ref?.deref(); // 可能已被 GC } }

📋 React/Vue 清单 框架最佳实践

**React useEffect 清理清单：** □ addEventListener → return 中 removeEventListener □ setInterval/setTimeout → return 中 clearInterval/clearTimeout □ fetch/axios → return 中 controller.abort() □ subscribe → return 中 unsubscribe □ IntersectionObserver/ResizeObserver → return 中 disconnect() **Vue onUnmounted 清理清单：** □ onMounted 中的 addEventListener → onUnmounted 中移除 □ watch/computed 中的副作用 → onUnmounted 中清理 □ 第三方库实例 → onUnmounted 中 destroy() □ 定时器 → onUnmounted 中清除
