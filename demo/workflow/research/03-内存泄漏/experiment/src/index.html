🔬 内存泄漏复现实验

泄漏 1: Detached DOM

每次点击创建 100 个 DOM 节点并从文档树移除，但 JS 仍持有引用。在 DevTools Memory 的 Heap Snapshot 中搜索 Detached 可看到泄漏节点。

已创建: 0 个 Detached 节点

泄漏 2: 事件监听器泄漏

每次点击添加 10 个事件监听器到 window，但不移除。Heap Snapshot 中搜索 EventListener 可看到数量增长。

已添加: 0 个未清理的监听器

泄漏 3: 闭包捕获泄漏

每次点击创建一个闭包，该闭包捕获了 1MB 的大字符串，但闭包本身只做简单打印。Heap Snapshot 中展开 Closure 对象的 [[Scopes]] 可看到被捕获的数据。

已创建: 0 个泄漏闭包 (累计约 0 MB)

✅ 修复方案

点击"修复"按钮后手动 GC，再拍 Heap Snapshot 对比，确认泄漏对象被回收。

注意：GC 时机由 V8 引擎决定，修复后可能需要等几秒才生效。
