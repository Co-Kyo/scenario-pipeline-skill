🔬 实验：内存泄漏 — 长时间运行页面的性能退化排查

包含 4 个独立实验，演示常见内存泄漏模式。建议配合 Chrome DevTools → Memory 面板使用。

实验 1：Map vs WeakMap 内存对比

强引用（Map）会阻止对象被 GC 回收，即使外部已无引用。WeakMap 的键是弱引用，对象可被自动回收。
操作：分别点击「用 Map 存储」和「用 WeakMap 存储」，然后「删除外部引用」，观察内存差异。

实验 2：事件监听器泄漏演示

组件切换时未移除事件监听器，导致旧监听器累积。每次切换都创建新监听器，旧的无法被 GC。
操作：反复点击「切换组件」，观察监听器计数持续增长。

✅ 检查点 2

泄漏模式下，反复切换后监听器数量 = 切换次数（每次 +1 不清理）。修复模式下，监听器数量保持为 1（先 remove 再 add）。

实验 3：setInterval 闭包引用泄漏

setInterval 回调闭包持有外部大对象引用，即使组件已「销毁」，定时器仍在运行，闭包中的对象无法释放。
操作：启动定时器 → 销毁组件 → 观察定时器仍在输出。

点击②后组件从 DOM 移除，但定时器仍每秒输出日志（闭包持有引用）。点击③后定时器停止，对象可被 GC 回收。观察内存数值变化。

实验 4：Detached DOM 检测

DOM 节点从文档树移除后，如果仍有 JS 变量引用它，它处于 Detached 状态（游离节点），无法被 GC。
操作：创建节点 → 移除 → 保持引用 → 观察 Detached 状态。

✅ 检查点 4

步骤②后节点从 DOM 消失，但 JS 数组仍持有引用 → Detached 计数 = 100。步骤③释放引用后 Detached 归零，GC 可回收。
验证：在 DevTools → Memory 拍堆快照，搜索 "Detached" 可找到游离节点。
