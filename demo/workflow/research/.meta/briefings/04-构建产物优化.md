# P4 构建产物优化 — 组装 Briefing

## 命题信息
命题：构建产物优化：Webpack/Vite 打包体积过大——首屏 JS 超过 500KB
通用占比：60-80%
限定词：Webpack/Vite

## 涉及能力摘要

### A6-HTTP 缓存策略 [用于: overview, edge-cases]
机制：强缓存 + 协商缓存两层。静态资源用 contenthash 文件名 + 长期缓存。
瓶颈：
- B1 缓存失效全量回源（状态跃迁，P0）：hash 配置错 → 用户看旧版或重下载
权衡：长期缓存+hash 文件名 vs no-cache 协商缓存

### A7-Code Splitting 与 Tree Shaking [用于: overview, edge-cases, trade-offs]
机制：动态 import() 生成独立 chunk。Tree Shaking 基于 ESM 删除未使用代码。sideEffects 标记。
瓶颈：
- B1 Tree Shaking 失效（输入变异，P0）：CommonJS 或 sideEffects 未配置
- B2 vendor chunk 膨胀（规模拐点，P1）：大库（moment.js+locales）污染 vendor
- B3 chunk 粒度太细（规模拐点，P1）：过度分割 → HTTP 请求数过多
权衡：路由级 vs 组件级分割；单一 vendor vs 拆分 vendor

## 内容比例约束
开篇 10-15%：从首屏 JS >500KB 导致 LCP 慢的痛点切入
主体 70-80%：Code Splitting 策略、Tree Shaking 原理、chunk 划分、持久化缓存等通用原理
收尾 10-15%：回到 Webpack 配置 / Vite 预构建给落地方案
