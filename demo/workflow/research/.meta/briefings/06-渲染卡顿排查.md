# P6 渲染卡顿排查 — 组装 Briefing

## 命题信息
命题：渲染卡顿排查：主线程长任务阻塞交互——动画掉帧/输入延迟
通用占比：≥ 80%
限定词：无

## 涉及能力摘要

### A1-浏览器渲染管线 [用于: overview, edge-cases, trade-offs]
机制：CRP：Layout → Paint → Composite。transform/opacity 最高效，width/height 最昂贵。
瓶颈：
- B1 强制同步布局（时序竞争，P0）：交替读写布局属性 → 帧率 15fps
- B2 触发布局属性的动画（时序竞争，P1）：width/height 动画 → 每帧 Layout+Paint
权衡：transform/opacity vs width/height 动画

### A2-事件循环机制 [用于: overview, edge-cases]
机制：微任务→rAF→渲染→宏任务。微任务可独占主线程。
瓶颈：
- B1 微任务风暴（时序竞争，P0）：Promise 链过长 → 页面卡死
- B2 宏任务饥饿（规模拐点，P1）：每任务 >16ms → 帧率 <60fps
权衡：同步批量 vs 切片执行

### A4-DevTools 性能分析 [用于: overview, edge-cases]
机制：Performance 面板识别 Long Task，Memory 面板定位泄漏。
瓶颈：
- B1 未识别 Long Task（资源边界，P0）：不知哪些代码阻塞主线程

### A8-Core Web Vitals 指标体系 [用于: overview, edge-cases]
机制：INP 衡量交互响应性（≤200ms），替代 FID。
瓶颈：
- B1 INP 响应延迟（时序竞争，P0）：事件处理 >200ms → 点击后卡顿

### A10-Web Worker 多线程 [用于: trade-offs]
机制：Worker 独立线程运行 JS，postMessage 通信。Transferable Objects 零拷贝。
瓶颈：
- B1 序列化开销（资源边界，P1）：传递大量数据 → 通信延迟>计算节省
权衡：postMessage vs SharedArrayBuffer

### A13-CSS contain 与合成层 [用于: edge-cases, trade-offs]
机制：contain 隔离子树。will-change 提升合成层。
瓶颈：
- B1 层爆炸（资源边界，P1）：大量 will-change → GPU 内存暴涨
权衡：contain:content vs contain:strict

### A16-Long Tasks API 与性能监测 [用于: overview]
机制：Long Tasks API 监测 >50ms 任务。PerformanceObserver 采集。
瓶颈：
- B1 Long Task 未被发现（资源边界，P0）：未部署监测 → 无告警

## 内容比例约束
开篇 10-15%：从动画掉帧/输入延迟的用户体感切入
主体 70-80%：事件循环机制、Long Task 定位、Web Worker 卸载、CSS contain 等通用原理
收尾 10-15%：回到 React Concurrent Mode / Vue nextTick 给落地方案
