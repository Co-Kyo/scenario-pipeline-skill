# P7 滚动性能 — 组装 Briefing

## 命题信息
命题：滚动性能：长页面大量图片的懒加载与滚动优化——电商商品列表
通用占比：≥ 80%
限定词：无

## 涉及能力摘要

### A1-浏览器渲染管线 [用于: overview, edge-cases]
机制：CRP 全链路。滚动时触发 Layout/Paint 是卡顿根源。
瓶颈：
- B1 强制同步布局（时序竞争，P0）：滚动回调中读写布局属性 → 帧率骤降

### A8-Core Web Vitals 指标体系 [用于: overview]
机制：CLS 衡量布局偏移（≤0.1），LCP 衡量加载性能。
瓶颈：
- B1 CLS 布局偏移（状态跃迁，P1）：图片加载导致元素位移 → CLS 增加

### A9-IntersectionObserver 与懒加载 [用于: overview, edge-cases, trade-offs]
机制：IO 异步观察元素可见性，rootMargin 提前加载。
瓶颈：
- B1 懒加载图片闪烁（状态跃迁，P0）：无占位 → CLS 增加
- B2 IO 回调阻塞（时序竞争，P1）：回调中重计算 → 主线程阻塞
权衡：IO vs scroll+getBoundingClientRect

### A12-图片格式与压缩策略 [用于: overview, edge-cases, trade-offs]
机制：WebP 比 JPEG 小 25-34%，AVIF 再小 20%。<picture> 多格式回退。
瓶颈：
- B1 格式兼容性（输入变异，P1）：AVIF 老浏览器不支持
- B2 图片过大未压缩（资源边界，P0）：未压缩原始图片 → LCP 慢
权衡：WebP vs AVIF；前端处理 vs CDN 动态处理

### A13-CSS contain 与合成层 [用于: edge-cases, trade-offs]
机制：contain 隔离子树。content-visibility:auto 跳过视口外渲染。
瓶颈：
- B1 层爆炸（资源边界，P1）：大量 will-change → GPU 内存暴涨
权衡：contain:content vs contain:strict

## 内容比例约束
开篇 10-15%：从电商商品列表滚动卡顿的体感切入
主体 70-80%：IntersectionObserver 懒加载、图片格式优化、contain 属性、合成层等通用原理
收尾 10-15%：回到 Vue/React 具体懒加载组件给落地方案
