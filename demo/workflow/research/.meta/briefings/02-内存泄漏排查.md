# P2 内存泄漏排查 — 组装 Briefing

## 命题信息
命题：内存泄漏排查：SPA 长时间运行后卡顿——事件监听/定时器/Detached DOM
通用占比：≥ 80%
限定词：无

## 涉及能力摘要

### A3-V8 GC 机制与堆内存管理 [用于: overview, edge-cases, trade-offs]
机制：分代 GC：新生代 Scavenge（半空间复制），老生代 Mark-Sweep-Compact（增量/并发标记）。对象两次 Scavenge 存活则晋升老生代。WeakRef/WeakMap 不阻止 GC。
瓶颈：
- B1 Detached DOM 节点（资源边界，P0）：DOM 移除但 JS 持有引用 → 内存持续增长
- B2 全局缓存无界增长（资源边界，P0）：Map/Set 无淘汰策略 → 老生代持续上升
- B3 GC 停顿掉帧（时序竞争，P1）：老生代 GC 标记清除 → 周期性帧率下降
权衡：强引用 Map（保证不回收）vs 弱引用 WeakMap（允许 GC 回收），缓存用 WeakMap

### A4-DevTools 性能分析 [用于: overview, edge-cases]
机制：Performance 面板录制时间线识别 Long Task。Memory 面板 Heap Snapshot 对比定位泄漏，Retainers 链追踪无法 GC 的原因。
瓶颈：
- B1 未识别 Long Task（资源边界，P0）：不知哪些代码阻塞主线程 → 无法定位
- B2 Detached DOM 未检测（资源边界，P0）：不知有 DOM 泄漏 → 内存持续增长
权衡：Performance 面板（精确需手动）vs Lighthouse（自动化但不够精确）

### A14-资源生命周期管理 [用于: overview, edge-cases, trade-offs]
机制：所有异步资源绑定与清理成对。AbortController 统一取消多个资源。
瓶颈：
- B1 事件监听未解绑（资源边界，P0）：SPA 路由切换未清理 → 内存增长+重复回调
- B2 Observer 未 disconnect（资源边界，P1）：组件销毁后 Observer 仍运行 → CPU 占用
- B3 请求未取消（资源边界，P1）：卸载后 fetch 响应仍回来 → state 更新警告
权衡：手动逐个清理（精确但易遗漏）vs AbortController 统一取消（简单但需封装）

## 内容比例约束
开篇 10-15%：从 SPA 长时间运行后越来越卡的用户反馈切入
主体 70-80%：V8 GC 原理、Heap Snapshot 分析、Detached DOM 定位、资源清理规范
收尾 10-15%：回到 React useEffect cleanup / Vue onUnmounted 给落地方案
