实验 1：Layout Thrashing — 读写交替 vs 批量读写

A3 重绘与回流

❌ 读写交替（强制同步 Layout）

每次：读 offsetHeight → 写 style.height → 触发 Layout → 循环

✅ 批量读 → 批量写（单次 Layout）

先读所有 offsetHeight → 再写所有 style.height → 仅 1 次 Layout
[A3] 原理：读 offsetHeight 等几何属性时，若 DOM 已被标记为 dirty， 浏览器被迫立即执行同步 Layout（Forced Synchronous Layout）。循环中反复"写-读-写"形成 Layout Thrashing，耗时可达批量方案的 10-100 倍。

实验 2：任务切片 — 同步长任务 vs rAF 时间分片

✅ rAF 时间分片（不阻塞渲染）

[A30+A31] 原理：同步长任务（macrotask）在执行期间完全阻塞主线程，浏览器无法执行 rAF 回调、处理输入或进行渲染。rAF 时间分片将任务拆成多帧执行，每帧只做预算时间内（如 8ms）的工作， 剩余时间让给渲染管线，保持 60fps 流畅。
