# A3 - 重绘与回流（Repaint vs Reflow）

## 概述

重绘（Repaint）和回流（Reflow/Layout）是浏览器渲染管线中最关键的两个性能敏感阶段。回流是重新计算元素的几何属性（位置和尺寸），重绘是重新绘制像素。回流**一定**引发重绘，但重绘**不一定**引发回流。优化这两者是前端性能调优的核心工作。

## 核心机制

### 回流（Reflow / Layout）

回流是浏览器重新计算渲染树中部分或全部节点的**位置和尺寸**的过程。

**触发回流的操作**：
- **几何属性变更**：`width`、`height`、`padding`、`margin`、`border`、`position`、`left`/`top`、`display`、`float`、`overflow`
- **DOM 结构变更**：添加/删除/移动 DOM 节点、改变元素内容（影响尺寸）
- **布局属性读取**：读取 `offsetWidth`、`offsetHeight`、`scrollTop`、`scrollLeft`、`getComputedStyle()`、`getBoundingClientRect()` 等会强制浏览器同步计算布局
- **视口变化**：窗口 resize、设备旋转
- **字体加载**：Web Font 加载完成可能导致文字重新排版

**回流的范围**：
- **全局回流**：根节点变化（如改变 `<html>` 的字体大小）影响整个页面
- **局部回流**：只影响变化节点及其子树
- 现代浏览器会将多个 DOM 变更**合并**到一个回流批次中（异步回流）
- 但读取布局属性会**强制同步回流**

### 重绘（Repaint）

重绘是在不改变布局的情况下，重新绘制受影响区域的**像素**。

**触发重绘但不触发回流的操作**：
- `color`、`background-color`、`visibility`
- `box-shadow`、`text-decoration`、`outline`
- `border-radius`（仅视觉变化，不影响布局时）
- `cursor`、`opacity`（在某些情况下）

**绘制层（Paint Layers）**：
- 浏览器将页面分为多个绘制层
- 每个层独立光栅化
- 重绘只影响包含变化元素的层

### 合成（Composite）

`transform` 和 `opacity` 变化**不触发回流也不触发重绘**，只触发合成：

```css
/* ✅ 只触发 Composite — 最轻量 */
.element {
  transform: translateX(100px);
  opacity: 0.5;
}

/* ❌ 触发 Reflow + Repaint — 最重量 */
.element {
  left: 100px;
}
```

### 浏览器的回流优化

现代浏览器采用**异步回流**策略：

1. 将 DOM 变更记录到一个队列中
2. 在下一帧（通常 16.6ms）统一执行回流
3. 合并多次变更为一次回流

但以下操作会**破坏**这种优化，强制**同步回流**：
- 读取布局属性：`offsetTop`、`scrollTop`、`clientWidth`、`getComputedStyle()`、`getBoundingClientRect()`
- 调用 `getComputedStyle()`

## 工程瓶颈

### 瓶颈1：布局抖动（Layout Thrashing）

- **触发条件**：JavaScript 在循环中交替读取布局属性和修改样式
- **症状**：帧率从 60fps 骤降到个位数，动画明显卡顿
- **检测**：Chrome DevTools Performance 面板中密集出现的 "Recalculate Style" 和 "Layout" 交替条带
- **缓解**：
  ```js
  // ❌ 布局抖动
  items.forEach(item => {
    const height = item.offsetHeight; // 强制同步回流
    item.style.height = (height + 10) + 'px'; // 修改样式
  });

  // ✅ 批量读写
  const heights = items.map(item => item.offsetHeight); // 批量读
  items.forEach((item, i) => {
    item.style.height = (heights[i] + 10) + 'px'; // 批量写
  });
  ```

### 瓶颈2：动画使用几何属性

- **触发条件**：使用 `left`、`top`、`width`、`height` 做动画
- **症状**：每帧触发 Reflow + Repaint，动画不流畅
- **检测**：Performance 面板中动画帧的 Layout 时间过长
- **缓解**：
  ```css
  /* ❌ 每帧 Reflow */
  .animate { animation: moveBad 1s infinite; }
  @keyframes moveBad { to { left: 200px; } }

  /* ✅ 只触发 Composite */
  .animate { animation: moveGood 1s infinite; }
  @keyframes moveGood { to { transform: translateX(200px); } }
  ```

### 瓶颈3：频繁插入 DOM 节点

- **触发条件**：循环中逐个 `appendChild`
- **症状**：每次插入触发一次 Reflow
- **检测**：Performance 面板中大量 Layout 事件
- **缓解**：DocumentFragment 批量插入；使用 `display: none` 离线操作后恢复

### 瓶颈4：CSS 选择器复杂度

- **触发条件**：深层嵌套选择器（如 `div > ul > li > a > span`）
- **症状**：Style Recalculation 耗时增加
- **检测**：Performance 面板 "Recalculate Style" 耗时
- **缓解**：使用简单类选择器、BEM 命名规范、减少选择器深度

### 瓶颈5：读取 getComputedStyle 触发同步回流

- **触发条件**：修改样式后立即调用 `getComputedStyle()` 或 `getBoundingClientRect()`
- **症状**：浏览器被迫中断优化，立即执行同步回流
- **检测**：Performance 面板中的 "Forced reflow" 警告
- **缓解**：缓存计算结果、批量读取、使用 ResizeObserver 替代手动读取

## 调试工具

| 工具 | 用途 |
|------|------|
| Chrome DevTools → Performance | 录制帧，分析 Layout/Recalculate Style/Paint 耗时 |
| Chrome DevTools → Rendering → Paint flashing | 绿色闪烁标记重绘区域 |
| Chrome DevTools → Rendering → Layout Shift Regions | 蓝色高亮布局偏移 |
| `getComputedStyle()` 前后的 Performance 标记 | 识别强制同步回流 |
| Layout Instability API | 监控 CLS（Cumulative Layout Shift） |
| Web Vitals 库 | `onCLS()` 监控布局偏移指标 |

## 典型权衡

### 权衡1：`will-change` 提示 vs 内存开销

- **使用 `will-change`**：提前告知浏览器该元素将变化，浏览器可预先创建合成层，避免回流
- **开销**：每个 `will-change` 元素会创建独立合成层，消耗 GPU 内存
- **实践**：动画前设置 `will-change: transform`，动画结束后移除

### 权衡2：`contain` 属性 vs 兼容性

- **使用 `contain: layout style paint`**：将元素隔离为独立的布局/绘制边界，限制回流传播范围
- **风险**：部分旧浏览器不支持，可能影响子元素的样式继承
- **实践**：在列表项、卡片组件等重复元素上使用

### 权衡3：减少 Reflow vs 代码可读性

- **极致优化**：将所有样式读取集中、所有修改集中，代码结构被性能需求扭曲
- **可读性优先**：代码逻辑清晰但可能有额外回流开销
- **实践**：在热路径（动画、滚动事件）中优先性能，普通逻辑中优先可读性

## 最小验证实验

### 实验：回流 vs 重绘 vs 合成 性能对比

```html
<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8">
  <title>Reflow vs Repaint vs Composite</title>
  <style>
    .box {
      width: 60px; height: 60px; margin: 5px;
      display: inline-block; background: #4CAF50;
    }
    .controls { margin: 20px 0; }
    button { padding: 8px 16px; margin: 4px; cursor: pointer; }
    #stats { margin: 10px; padding: 10px; background: #f5f5f5; font-family: monospace; }
  </style>
</head>
<body>
  <h1>Reflow vs Repaint vs Composite</h1>
  <div class="controls">
    <button onclick="animateReflow()">回流动画 (left)</button>
    <button onclick="animateComposite()">合成动画 (transform)</button>
    <button onclick="triggerLayoutThrash()">布局抖动测试</button>
    <button onclick="batchOperation()">批量操作</button>
    <button onclick="resetAll()">重置</button>
  </div>
  <div id="stats"></div>
  <div id="container"></div>

  <script>
    const container = document.getElementById('container');
    const stats = document.getElementById('stats');
    const boxes = [];

    // 创建 200 个方块
    for (let i = 0; i < 200; i++) {
      const div = document.createElement('div');
      div.className = 'box';
      div.textContent = i;
      container.appendChild(div);
      boxes.push(div);
    }

    let animId = null;

    function animateReflow() {
      cancelAnimationFrame(animId);
      let offset = 0;
      function frame() {
        offset += 2;
        if (offset > 200) offset = 0;
        boxes.forEach(b => {
          b.style.position = 'relative';
          b.style.left = offset + 'px'; // ❌ 触发 Reflow
        });
        stats.textContent = `Reflow动画: offset=${offset}`;
        animId = requestAnimationFrame(frame);
      }
      frame();
    }

    function animateComposite() {
      cancelAnimationFrame(animId);
      let offset = 0;
      function frame() {
        offset += 2;
        if (offset > 200) offset = 0;
        boxes.forEach(b => {
          b.style.transform = `translateX(${offset}px)`; // ✅ 只触发 Composite
        });
        stats.textContent = `Composite动画: offset=${offset}`;
        animId = requestAnimationFrame(frame);
      }
      frame();
    }

    function triggerLayoutThrash() {
      cancelAnimationFrame(animId);
      const start = performance.now();
      // ❌ 布局抖动：交替读写
      boxes.forEach(b => {
        b.style.width = (b.offsetWidth + 1) + 'px';
        b.style.height = (b.offsetHeight + 1) + 'px';
      });
      const time = performance.now() - start;
      stats.textContent = `布局抖动耗时: ${time.toFixed(2)}ms (观察Performance面板中的Layout事件)`;
    }

    function batchOperation() {
      cancelAnimationFrame(animId);
      const start = performance.now();
      // ✅ 批量读
      const dims = boxes.map(b => ({
        w: b.offsetWidth,
        h: b.offsetHeight
      }));
      // ✅ 批量写
      boxes.forEach((b, i) => {
        b.style.width = (dims[i].w + 1) + 'px';
        b.style.height = (dims[i].h + 1) + 'px';
      });
      const time = performance.now() - start;
      stats.textContent = `批量操作耗时: ${time.toFixed(2)}ms`;
    }

    function resetAll() {
      cancelAnimationFrame(animId);
      boxes.forEach(b => {
        b.style.cssText = '';
      });
      stats.textContent = '已重置';
    }
  </script>
</body>
</html>
```

**实验步骤**：
1. 打开 Chrome DevTools → Performance，开始录制
2. 分别点击 "回流动画" 和 "合成动画"，观察帧率差异
3. 开启 Rendering → Paint flashing，观察回流动画的绿色闪烁（合成动画无闪烁）
4. 对比 "布局抖动" 和 "批量操作" 的耗时
5. 在 Performance 火焰图中观察 Layout 事件的数量和耗时
