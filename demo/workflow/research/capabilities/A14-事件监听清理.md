# A14 - 事件监听清理

## 核心机制

### addEventListener / removeEventListener 基本机制

`addEventListener(type, listener, options)` 将回调函数注册到目标元素的事件监听器列表中。浏览器内部维护一个 **Listener Map**：`{ eventType → [listener1, listener2, ...] }`。

**关键特性**：
- 同一元素同一事件类型可注册多个监听器
- 相同的 `(type, listener, capture)` 三元组视为同一个监听器，不会重复注册
- 监听器引用了回调函数及其闭包上下文，阻止 GC 回收

### listener 匹配规则

```javascript
// 以下两者是不同的监听器（capture 值不同）
el.addEventListener('click', handler, true);
el.addEventListener('click', handler, false); // 不会覆盖上面的

// 匿名函数无法移除（每次创建新函数引用）
el.addEventListener('click', () => console.log('hi'));
el.removeEventListener('click', () => console.log('hi')); // 无效！
```

### Options 对象

```javascript
el.addEventListener('click', handler, {
  capture: false,   // 是否在捕获阶段触发
  once: true,       // 触发一次后自动移除
  passive: true,    // 不会调用 preventDefault()
  signal: controller.signal  // AbortSignal，调用 abort() 自动移除
});
```

### AbortController 用于批量清理

```javascript
const controller = new AbortController();

el1.addEventListener('click', handler1, { signal: controller.signal });
el2.addEventListener('scroll', handler2, { signal: controller.signal });
el3.addEventListener('resize', handler3, { signal: controller.signal });

// 一次性移除所有通过该 signal 注册的监听器
controller.abort();
```

### 自动清理场景

以下情况监听器会自动移除：
1. **元素从 DOM 移除**：监听器随元素被 GC 回收（前提是没有其他 JS 引用）
2. **`once: true`**：触发一次后自动移除
3. **AbortSignal abort**：通过 signal 参数注册的监听器在 abort 时移除
4. **文档导航**：页面导航时旧文档的监听器被清理

## 工程瓶颈

1. **匿名函数监听器**：无法通过 `removeEventListener` 移除，是内存泄漏的主要来源。
2. **闭包持有大对象**：监听器回调闭包可能意外持有大数组、DOM 引用等，阻止 GC。
3. **SPA 路由切换**：组件卸载时未清理在全局对象（window、document）上注册的监听器。
4. **第三方库泄漏**：某些库在内部注册监听器但不提供清理接口。
5. **事件委托 vs 直接绑定**：在大量子元素上直接绑定监听器造成内存膨胀。
6. **Passive 误解**：错误地在 passive listener 中调用 `preventDefault()` 导致无效操作。
7. **跨 iframe 通信**：`message` 事件监听器在 iframe 销毁后可能残留。

## 调试工具

| 工具 | 用途 |
|------|------|
| Chrome DevTools → Elements → Event Listeners | 查看元素上注册的所有监听器 |
| Chrome DevTools → Memory → Heap Snapshot | 搜索 `EventListener` 或 `listener` 定位泄漏 |
| `getEventListeners(el)` (Console) | 获取元素上的所有监听器（仅 DevTools Console） |
| `monitorEvents(el)` / `unmonitorEvents(el)` | 监控元素事件触发 |
| Chrome DevTools → Performance 面板 | 观察事件处理对帧率的影响 |
| ESLint `no-unused-vars` | 检测未使用的事件处理函数变量 |

## 典型权衡

| 维度 | 权衡 |
|------|------|
| 事件委托 vs 直接绑定 | 委托更省内存但增加事件冒泡开销；直接绑定内存多但触发更快 |
| 匿名函数 vs 命名函数 | 匿名函数代码更简洁但无法移除；命名函数可移除但增加代码量 |
| passive: true | 提升滚动性能，但无法调用 preventDefault() |
| AbortController | 统一清理方便，但需额外管理 controller 生命周期 |
| 全局监听器 vs 局部监听器 | 全局（window/document）需手动清理；局部随元素 GC |

## 最小验证实验

### 实验 1：匿名函数泄漏

```javascript
// 1. 拍摄 Heap Snapshot 1
// 2. 执行
function setupListeners() {
  for (let i = 0; i < 1000; i++) {
    const div = document.createElement('div');
    document.body.appendChild(div);
    // 匿名函数 → 无法 removeEventListener
    div.addEventListener('click', () => {
      console.log(div.textContent);
    });
    document.body.removeChild(div);
  }
}
setupListeners();

// 3. GC → 拍摄 Heap Snapshot 2
// 4. Comparison 视图：观察 detached div 数量和 EventListener 数量
```

### 实验 2：AbortController 清理

```javascript
const controller = new AbortController();

function setup() {
  window.addEventListener('scroll', onScroll, { signal: controller.signal });
  window.addEventListener('resize', onResize, { signal: controller.signal });
  document.addEventListener('keydown', onKey, { signal: controller.signal });
}

function teardown() {
  controller.abort(); // 一次性清理全部
}

setup();
// ... 使用后
teardown();
```

### 实验 3：once 选项

```javascript
let count = 0;
button.addEventListener('click', () => {
  count++;
  console.log(`Clicked ${count} times`);
}, { once: true });
// 点击一次后自动移除
```
