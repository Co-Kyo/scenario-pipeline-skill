# A26 - 图片懒加载（loading=lazy）

## 核心机制

### 原生 loading="lazy"
- HTML 属性，直接加在 `<img>` 或 `<iframe>` 上
- 浏览器自行决定何时加载：通常在距 viewport 约 1250px（Chrome）时触发预加载
- 无需 JavaScript，零运行时开销
- `loading="eager"`（默认）立即加载，`loading="lazy"` 延迟加载
- 浏览器支持：Chrome 77+、Firefox 75+、Safari 15.4+、Edge 79+

### IntersectionObserver 实现
- 通过 `data-src` 存储真实 URL，`src` 放占位图或空
- 创建 IntersectionObserver 监听元素进入 viewport
- 进入时替换 `data-src` → `src`，触发图片加载
- 支持自定义 rootMargin（提前加载距离）和 threshold
- 可精细控制加载时机、添加动画效果

### 关键差异

| 维度 | loading="lazy" | IntersectionObserver |
|------|---------------|---------------------|
| 实现复杂度 | 零代码 | 需要 JS 逻辑 |
| 浏览器优化 | 浏览器内置启发式 | 依赖 JS 执行时机 |
| 首屏图片 | 浏览器自动识别首屏不延迟 | 需手动排除首屏 |
| CLS 控制 | 需配合 width/height | 可自定义占位尺寸 |
| iframe 支持 | ✓ | 需额外实现 |
| 自定义程度 | 低 | 高（动画、优先级等） |

## 工程瓶颈

1. **首屏 LCP 图片误懒加载**：浏览器对 viewport 内图片的启发式判断不完美，Lighthouse 会对首屏 lazy 图片发出警告
2. **CLS 问题**：懒加载图片若未预设 width/height，加载时会导致布局偏移
3. **SEO 影响**：Googlebot 需要 JavaScript 执行 IntersectionObserver，原生 loading="lazy" 对爬虫更友好
4. **Scroll 事件回退性能**：IntersectionObserver 不支持时回退到 scroll 事件监听，需要 throttling/debouncing
5. **图片预加载距离不可控**：原生 lazy 的预加载距离（~1250px）无法自定义，某些场景可能不够

## 调试工具

- **Chrome DevTools → Network**：滚动页面观察图片请求时机
- **Lighthouse**：检测首屏图片是否被错误懒加载
- **Performance 面板**：观察图片加载与 scroll 事件的时间关系
- **Chrome DevTools → Rendering → Layout Shift Regions**：可视化 CLS

## 典型权衡

1. **简单性 vs 控制力**：原生 loading="lazy" 零代码但不可自定义；IntersectionObserver 灵活但需维护 JS 代码
2. **带宽节省 vs LCP 风险**：过度懒加载首屏图片会严重损害 LCP，需对 above-the-fold 图片显式设置 `loading="eager"` 或 `fetchpriority="high"`
3. **SEO 友好 vs 动画效果**：原生方案对搜索引擎更友好；IntersectionObserver 方案可实现 fade-in 等加载动画提升体验

## 最小验证实验

```html
<!-- 原生方案 -->
<img src="photo.jpg" loading="lazy" alt="Lazy demo"
     width="800" height="600">

<!-- IntersectionObserver 方案 -->
<img class="lazy" data-src="photo.jpg" src="placeholder.svg"
     alt="Lazy demo" width="800" height="600">

<script>
const observer = new IntersectionObserver((entries) => {
  entries.forEach(entry => {
    if (entry.isIntersecting) {
      const img = entry.target;
      img.src = img.dataset.src;
      img.classList.remove('lazy');
      observer.unobserve(img);
    }
  });
}, { rootMargin: '200px' });

document.querySelectorAll('.lazy').forEach(img => observer.observe(img));
</script>
```

验证步骤：
1. 创建一个长页面（>10 张图片）
2. 打开 Chrome DevTools Network 面板
3. 滚动页面，观察图片请求是否在滚动到附近时才发出
4. 对比首屏图片（应立即加载）和非首屏图片（应延迟加载）的请求时机
