# A25 - 响应式图片（srcset/sizes）

## 核心机制

### srcset 属性
- 放在 `<img>` 或 `<source>` 上，提供多个候选图片 URL
- **w 描述符**：`srcset="img-300.jpg 300w, img-600.jpg 600w, img-1200.jpg 1200w"` — 声明图片固有宽度（像素），浏览器根据 viewport 宽度和 DPR 选择
- **x 描述符**：`srcset="icon.png 1x, icon@2x.png 2x"` — 按设备像素比选择，适用于固定尺寸图片

### sizes 属性
- 配合 srcset w 描述符使用，告诉浏览器图片在不同 viewport 下的显示宽度
- `sizes="(max-width: 600px) 100vw, (max-width: 1200px) 50vw, 600px"`
- 浏览器计算公式：`最终选择 = srcset 中最接近 (sizes指定宽度 × devicePixelRatio) 的图片`

### picture 元素
- `<picture>` 包含多个 `<source>` + 一个 `<img>` fallback
- `<source>` 支持 `media`（媒体查询）、`type`（MIME 类型）、`srcset`/`sizes`
- 用途：**艺术指导**（不同裁切）、**格式降级**（AVIF→WebP→JPEG）、**响应式尺寸**

### 浏览器选择算法
1. 浏览器解析 sizes 得到当前显示宽度
2. 乘以 devicePixelRatio 得到"理想像素密度"
3. 从 srcset 中选择 ≥ 理想像素密度的最小图片（或最大的，如果都不够）

## 工程瓶颈

1. **sizes 值不准确**：如果 sizes 与实际 CSS 布局不一致，浏览器会选择错误的图片尺寸，导致浪费带宽或图片模糊
2. **图片数量爆炸**：为每张图片生成多尺寸（300w/600w/900w/1200w/1800w）× 多格式（AVIF/WebP/JPEG）= 15 个文件
3. **缓存效率降低**：同一图片多版本分散缓存命中率
4. **构建复杂度**：需要 image processing pipeline（sharp、ImageMagick、CDN 动态裁切）
5. **LCP 图片特殊处理**：首屏 LCP 图片不应用 sizes 过小的值，否则加载低质量图片影响 Core Web Vitals

## 调试工具

- **Chrome DevTools → Elements → Computed**：查看浏览器实际选择的图片 URL
- **Chrome DevTools → Network → Img 过滤器**：查看所有图片请求及大小
- **Lighthouse**：Properly size images audit
- **Responsive Image Linter**：检查 srcset/sizes 配置一致性
- **Picturefill**（polyfill）：为不支持 `<picture>` 的旧浏览器提供支持

## 典型权衡

1. **精确性 vs 维护成本**：sizes 精确匹配 CSS 布局可最大化带宽节省，但 CSS 变动时需同步更新 sizes 属性
2. **图片数量 vs CDN 动态裁切**：预生成多尺寸增加构建时间和存储；CDN 实时裁切（如 Cloudflare Image Resizing）减少存储但增加首次请求延迟
3. **Art Direction vs 纯尺寸缩放**：`<picture>` + media 查询做艺术指导（不同裁切）增加设计工作量，但可显著提升移动端视觉体验

## 最小验证实验

```html
<img
  srcset="photo-400.jpg 400w,
          photo-800.jpg 800w,
          photo-1200.jpg 1200w"
  sizes="(max-width: 600px) 100vw,
         (max-width: 1000px) 50vw,
         800px"
  src="photo-800.jpg"
  alt="Responsive demo"
  width="1200" height="800"
>
```

验证步骤：
1. 准备 3 张不同尺寸的图片（400px/800px/1200px 宽）
2. 在 Chrome DevTools 中模拟不同 viewport 宽度和 DPR
3. 观察 Network 面板确认浏览器选择了正确的图片
4. 在移动端真机上验证实际加载的图片大小
