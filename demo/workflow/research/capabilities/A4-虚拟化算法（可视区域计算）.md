# A4 - 虚拟化算法（可视区域计算）

## 核心机制

虚拟列表（Virtual List / Virtualized List）是一种通过仅渲染可视区域内的 DOM 节点来优化大数据量列表渲染性能的技术。其核心思想是：**不为每一条数据创建 DOM 节点，而是根据滚动位置动态计算出需要渲染的子集**。

### 1. 可视区域计算

可视区域计算是虚拟列表的基础。核心公式：

```
startIndex = Math.floor(scrollTop / itemHeight)
endIndex = startIndex + Math.ceil(containerHeight / itemHeight)
```

- `scrollTop`：容器当前滚动距离
- `itemHeight`：单个列表项高度
- `containerHeight`：可视容器高度

对于**动态高度**场景，需要维护一个高度映射表（heightMap），通过预估高度（estimatedItemHeight）填充未测量项，已渲染项通过 `offsetHeight` 或 `ResizeObserver` 实测后回写。

### 2. 缓冲区（Overscan）

为避免快速滚动时出现白屏，需在可视区域上下各增加若干预渲染项：

```
visibleStartIndex = Math.max(0, startIndex - overscan)
visibleEndIndex = Math.min(totalCount - 1, endIndex + overscan)
```

`overscan` 通常设置为可视项数的 1-3 倍。过大会增加 DOM 节点数，过小会出现滚动白屏。

### 3. DOM 复用池（Recycler Pool）

DOM 复用池避免频繁的创建/销毁 DOM 节点。核心机制：

- **归还**：当列表项滚出可视区域时，将其 DOM 节点从容器移除并放入复用池
- **借出**：当需要新列表项时，优先从复用池取用，仅在池空时才创建新节点
- **重置**：借出时重置节点内容、样式、事件绑定

### 4. 容器高度占位

为保持正确的滚动条行为，需要一个占位元素撑起总高度：

```
totalHeight = sum(allItemHeights)  // 动态高度
totalHeight = itemCount * itemHeight  // 固定高度
```

占位元素通常使用 `height` + `transform: translateY()` 或绝对定位的 padding 元素。

## 工程瓶颈

### 瓶颈1：动态高度计算的精度与性能

- **触发条件**：列表项高度不固定，需要实时测量
- **症状**：滚动时出现跳动、闪烁；滚动条位置不准确
- **检测**：使用 Performance 面板观察 Layout/Recalculate Style 耗时；对比滚动帧率
- **缓解**：
  - 使用 `ResizeObserver` 替代 `offsetHeight` 读取（避免强制同步布局）
  - 缓存已测量高度，使用二分查找定位 startIndex
  - 预估高度 + 渐进修正策略

### 瓶颈2：快速滚动白屏

- **触发条件**：用户快速拖拽滚动条，overscan 不足
- **症状**：可见区域出现短暂空白
- **检测**：Performance 录制中观察帧间隔和 DOM 操作
- **缓解**：增大 overscan 值；使用 `will-change: transform` 启用 GPU 加速；考虑使用 `IntersectionObserver` 做预加载

### 瓶颈3：DOM 复用池的内存泄漏

- **触发条件**：复用池无限增长，或节点重置不彻底
- **症状**：内存持续增长；复用节点残留旧数据或事件监听
- **检测**：Memory 面板 Heap Snapshot 对比；检查 detached DOM 节点
- **缓解**：限制复用池最大容量；标准化节点重置流程；使用 `WeakMap` 关联节点元数据

### 瓶颈4：滚动事件节流与渲染同步

- **触发条件**：高频 scroll 事件触发重计算和 DOM 更新
- **症状**：滚动卡顿，帧率下降
- **检测**：Performance 面板火焰图中 scroll handler 耗时 > 4ms
- **缓解**：使用 `requestAnimationFrame` 节流滚动回调；将计算与 DOM 更新放在同一帧内完成

### 瓶颈5：列表项内嵌套复杂组件

- **触发条件**：列表项包含图片、视频、复杂组件
- **症状**：即使虚拟化，单个列表项渲染仍耗时过长
- **检测**：Performance 面板单帧内 Long Task
- **缓解**：列表项内部再做懒加载；使用 `React.memo` / `shouldComponentUpdate` 避免无效重渲染；图片使用 `loading="lazy"`

## 典型权衡

### 权衡1：固定高度 vs 动态高度

| 维度 | 固定高度 | 动态高度 |
|------|----------|----------|
| 计算复杂度 | O(1) 定位 | O(log n) 二分查找 + 高度缓存 |
| 滚动精度 | 完美 | 需要渐进修正，可能出现跳动 |
| 内存开销 | 无额外存储 | 需要 heightMap |
| **建议** | 优先选择固定高度；动态高度仅在业务必须时使用 |

### 权衡2：overscan 大小

| 维度 | 小 overscan (1-2) | 大 overscan (5-10) |
|------|-------------------|---------------------|
| DOM 节点数 | 少，内存低 | 多，内存高 |
| 滚动白屏风险 | 高 | 低 |
| 首次渲染开销 | 低 | 高 |
| **建议** | 根据列表项渲染复杂度和滚动速度动态调整 |

### 权衡3：DOM 复用 vs 每次创建

| 维度 | DOM 复用 | 每次创建/销毁 |
|------|----------|---------------|
| GC 压力 | 低 | 高 |
| 代码复杂度 | 高（需管理复用池） | 低 |
| 内存 | 复用池常驻 | 按需分配 |
| **建议** | 列表项渲染成本高时使用复用；简单列表可不使用 |

## 最小验证实验

```html
<!DOCTYPE html>
<html>
<head>
<style>
  * { margin: 0; padding: 0; box-sizing: border-box; }
  #container {
    height: 400px;
    overflow-y: auto;
    border: 1px solid #ccc;
    position: relative;
  }
  #phantom { position: relative; }
  #list {
    position: absolute;
    top: 0;
    left: 0;
    width: 100%;
  }
  .item {
    height: 40px;
    padding: 8px 16px;
    border-bottom: 1px solid #eee;
    display: flex;
    align-items: center;
  }
  .item:nth-child(even) { background: #f9f9f9; }
</style>
</head>
<body>
<h3>虚拟列表 Demo (100,000 项)</h3>
<div id="container">
  <div id="phantom">
    <div id="list"></div>
  </div>
</div>
<script>
const TOTAL = 100000;
const ITEM_HEIGHT = 40;
const CONTAINER_HEIGHT = 400;
const OVERSCAN = 5;

const container = document.getElementById('container');
const phantom = document.getElementById('phantom');
const list = document.getElementById('list');

// 设置占位高度
phantom.style.height = `${TOTAL * ITEM_HEIGHT}px`;

// DOM 复用池
const pool = [];
function acquireNode() {
  return pool.length > 0 ? pool.pop() : document.createElement('div');
}
function releaseNode(node) {
  pool.push(node);
}

let prevStart = -1;
let prevEnd = -1;

function render() {
  const scrollTop = container.scrollTop;
  const startIdx = Math.max(0, Math.floor(scrollTop / ITEM_HEIGHT) - OVERSCAN);
  const endIdx = Math.min(TOTAL - 1, Math.ceil((scrollTop + CONTAINER_HEIGHT) / ITEM_HEIGHT) + OVERSCAN);

  if (startIdx === prevStart && endIdx === prevEnd) return;
  prevStart = startIdx;
  prevEnd = endIdx;

  // 回收离开可视区的节点
  const children = Array.from(list.children);
  children.forEach(child => {
    const idx = parseInt(child.dataset.index);
    if (idx < startIdx || idx > endIdx) {
      list.removeChild(child);
      releaseNode(child);
    }
  });

  // 渲染新进入的节点
  for (let i = startIdx; i <= endIdx; i++) {
    if (!list.querySelector(`[data-index="${i}"]`)) {
      const node = acquireNode();
      node.className = 'item';
      node.dataset.index = i;
      node.style.position = 'absolute';
      node.style.top = `${i * ITEM_HEIGHT}px`;
      node.style.height = `${ITEM_HEIGHT}px`;
      node.style.width = '100%';
      node.textContent = `Item #${i}`;
      list.appendChild(node);
    }
  }
}

// 使用 rAF 节流
let ticking = false;
container.addEventListener('scroll', () => {
  if (!ticking) {
    requestAnimationFrame(() => {
      render();
      ticking = false;
    });
    ticking = true;
  }
});

render(); // 首次渲染
</script>
</body>
</html>
```
