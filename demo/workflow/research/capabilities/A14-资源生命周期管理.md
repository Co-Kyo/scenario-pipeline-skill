# 资源生命周期管理

> ID: A14 | 扇出: 2/8 | 耦合度: 1 | 战略价值: 1.0 | 🏕️ 三级能力

## 核心机制

所有异步资源（事件监听、定时器、Observer、请求、Worker、WebSocket）必须"绑定与清理成对"：

| 资源类型 | 绑定 | 清理 |
|---------|------|------|
| 事件监听 | `addEventListener` | `removeEventListener` |
| 定时器 | `setInterval` / `setTimeout` | `clearInterval` / `clearTimeout` |
| IntersectionObserver | `observe()` | `disconnect()` |
| MutationObserver | `observe()` | `disconnect()` |
| ResizeObserver | `observe()` | `disconnect()` |
| Fetch 请求 | `fetch()` | `AbortController.abort()` |
| WebSocket | `new WebSocket()` | `close()` + 置 null |
| Worker | `new Worker()` | `terminate()` |

**AbortController**：统一取消机制，一个 controller 可取消多个请求/监听：

```javascript
const ctrl = new AbortController();
fetch('/api', { signal: ctrl.signal });
window.addEventListener('resize', handler, { signal: ctrl.signal });
// 清理：ctrl.abort() 取消所有
```

## 工程瓶颈

| # | 瓶颈名 | 触发条件 | 表现症状 | 检测手段 | 缓解策略 |
|---|--------|---------|---------|---------|---------|
| 1 | 事件监听未解绑 | SPA 路由切换时未清理上一页的监听 | 内存增长 + 重复回调 | Memory 面板 + 事件计数 | useEffect cleanup 中解绑 |
| 2 | Observer 未 disconnect | 组件销毁后 MutationObserver/ResizeObserver 仍运行 | CPU 持续占用 | Performance 面板 | onUnmounted 中 disconnect |
| 3 | 请求未取消 | 组件卸载后 fetch 响应仍回来更新已卸载组件 | React state 更新警告 + 潜在内存泄漏 | 控制台 warning | AbortController |

## 典型权衡

| 维度 | 方案 A | 方案 B | 选择建议 |
|------|--------|--------|---------|
| 清理模式 | 手动逐个清理（精确但易遗漏）| AbortController 统一取消（简单但需封装）| 多个资源用 AbortController，单个手动清理 |
