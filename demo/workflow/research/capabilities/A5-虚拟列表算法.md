# 虚拟列表算法

> ID: A5 | 扇出: 2/8 | 耦合度: 1 | 战略价值: 1.0 | 🏕️ 三级能力

## 核心机制

虚拟列表（Virtual List / Windowing）的核心思想：**只渲染可视区域内的 DOM 节点**，通过滚动位置动态计算哪些项应该渲染。

**核心算法**：
1. **可视区域计算**：`containerHeight / itemHeight = 可见项数`
2. **滚动偏移映射**：`scrollTop → 起始索引 = Math.floor(scrollTop / itemHeight)`
3. **缓冲区**：可视区域上下各多渲染 N 个项（避免快速滚动白屏）
4. **DOM 池复用**：滚动时复用已离开可视区域的 DOM 节点（修改内容而非创建/销毁）

**动态高度处理**（难点）：
- 预估高度 → 渲染后测量真实高度 → 更新缓存
- 前缀和数组 + 二分查找定位：`startIndex = binarySearch(prefixSums, scrollTop)`
- 滚动锚定：高度变化时调整 scrollTop 防止跳动

## 工程瓶颈

| # | 瓶颈名 | 触发条件 | 表现症状 | 检测手段 | 缓解策略 |
|---|--------|---------|---------|---------|---------|
| 1 | 动态高度导致滚动跳动 | 项高度在渲染后才确定，更新缓存时触发重算 | 滚动位置突然偏移 | 用户反馈 + Performance 面板 | 预估高度 + 渐进修正 + 滚动锚定补偿 |
| 2 | 快速滚动白屏 | 缓冲区太小，快速滚动时新项未渲染完成 | 短暂空白区域 | 视觉观察 | 增大缓冲区（可视区域的 50%-100%）|
| 3 | 复杂列表项重渲染 | 列表项包含图片/组件，切换内容时仍然触发完整重渲染 | 滚动时 CPU 飙高 | Performance 面板 → 组件渲染 | 用 React.memo / Vue v-memo 缓存列表项 |
| 4 | 滚动事件频率过高 | 滚动事件每 16ms 触发一次 | 主线程被滚动回调占用 | Performance 面板 → scroll 事件 | 节流（requestAnimationFrame）或 IntersectionObserver |

## 调试工具

| 工具 | 用法 |
|------|------|
| Elements 面板 | 检查实际渲染的 DOM 节点数（应远小于总数据量）|
| Performance 面板 | 录制滚动操作，查看 Layout/Paint 耗时 |
| React DevTools Profiler | 查看哪些组件在滚动时重渲染 |

## 典型权衡

| 维度 | 方案 A | 方案 B | 选择建议 |
|------|--------|--------|---------|
| 高度策略 | 固定高度（算法简单，O(1) 定位）| 动态高度（灵活，但需前缀和+二分）| 列表项高度一致用固定，高度差异大用动态 |
| 实现方式 | 自己实现（完全可控）| react-virtualized / @tanstack/virtual（开箱即用）| 快速上手用库，深度定制自己实现 |
