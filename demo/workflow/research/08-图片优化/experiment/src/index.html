💡 格式选择决策树

// 最佳实践：<picture> 多格式回退 <picture> <source srcset="hero.avif" type="image/avif"> ← 优先 AVIF（最小） <source srcset="hero.webp" type="image/webp"> ← 回退 WebP（兼容好） <img src="hero.jpg" alt="Hero" ← 兜底 JPEG（100% 兼容） width="1920" height="1080"> ← 设尺寸避免 CLS </picture> // CDN 动态处理（推荐） // 只需上传原图，CDN 根据 Accept 头自动返回最优格式 <img src="https://cdn.example.com/hero?w=800&q=80&fmt=auto" alt="Hero"> // srcset 响应式 <img srcset="hero-480.webp 480w, hero-800.webp 800w, hero-1200.webp 1200w" sizes="(max-width: 600px) 480px, (max-width: 1000px) 800px, 1200px" src="hero-800.webp" alt="Hero">

⚠️ sizes 配置错误示例

// ❌ 错误：sizes 写死 100vw <img srcset="small.webp 480w, large.webp 1200w" sizes="100vw"> // 问题：桌面端也会加载 large.webp（1200w） // 但实际显示区域可能只有 400px // ✅ 正确：sizes 匹配实际显示宽度 <img srcset="small.webp 480w, large.webp 1200w" sizes="(max-width: 600px) 100vw, 50vw"> // 手机端：100vw → 选 small.webp (480w) // 桌面端：50vw → 选 small.webp (600px < 800w) // ❌ 错误：忘记设 width/height <img src="hero.webp" alt="Hero"> // 图片加载前浏览器不知道尺寸 → CLS! // ✅ 正确：始终设 width/height <img src="hero.webp" alt="Hero" width="800" height="600"> // 浏览器提前预留空间 → CLS = 0
