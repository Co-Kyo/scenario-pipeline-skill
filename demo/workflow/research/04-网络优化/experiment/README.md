# 网络优化：弱网环境下的资源加载策略 — 实验

## 概述

本实验通过纯前端模拟（无需服务端），演示四种核心网络优化策略在弱网环境下的行为差异，帮助理解各策略的适用场景和效果。

## 实验模块

### 实验 1：HTTP 缓存行为模拟

对比四种缓存策略的表现：

| 策略 | 行为 | 适用场景 |
|------|------|----------|
| **强缓存** `Cache-Control: max-age` | 命中时 **0ms** 响应，完全不发请求 | 静态资源（JS/CSS/图片）带 hash |
| **协商缓存** `ETag / If-None-Match` | 发请求验证，304 时复用缓存 | 频繁变更但内容稳定的 API |
| **no-cache** | 强制协商验证，但允许缓存存储 | 需要最新版本的 HTML 入口 |
| **no-store** | 完全不缓存，每次都完整传输 | 敏感数据（支付、密码页面） |

### 实验 2：指数退避重试

可调参数：
- **初始延迟**：首次重试前等待时间（默认 200ms）
- **最大重试次数**：放弃前的重试上限（默认 7 次）
- **退避因子**：每次延迟的倍增系数（默认 2，即 200→400→800→1600...）
- **抖动范围**：±百分比，防止多个客户端同时重试造成"惊群效应"

### 实验 3：资源压缩对比

使用 `TextEncoder` 计算原始体积，按各算法的模拟压缩率计算压缩后大小：

| 算法 | 典型压缩率 | 压缩速度 | 适用场景 |
|------|-----------|---------|----------|
| **gzip** | ~65-80% | 快 | 通用兼容，CPU 成本低 |
| **brotli** | ~75-90% | 较慢 | 静态资源预压缩，更优压缩比 |

### 实验 4：弱网模拟 & 瀑布图

用 `setTimeout` 模拟不同网络的 RTT 延迟：

| 网络 | RTT | 带宽 | 典型体验 |
|------|-----|------|----------|
| **WiFi** | ~5ms | 50 MB/s | 秒开 |
| **4G** | ~50ms | 5 MB/s | 快速 |
| **3G** | ~200ms | 500 KB/s | 可感知延迟 |
| **2G** | ~800ms | 50 KB/s | 明显卡顿 |

瀑布图展示了 DNS → TCP → TLS → 等待 → 下载 各阶段的时间占比。

## 验证检查点

| 编号 | 检查内容 | 验证方式 |
|------|----------|----------|
| CP1 | 强缓存命中时响应时间为 0ms | 自动测试强缓存第二次请求 |
| CP2 | 协商缓存命中返回 304 | 自动测试协商缓存序列 |
| CP3 | 退避间隔递增（允许 ±10% 抖动） | 运行模拟后自动校验 |
| CP4 | brotli 压缩率优于 gzip | 压缩对比后自动校验 |
| CP5 | 2G 总耗时 > 4G 的 3 倍 | 四种网络对比后自动校验 |
