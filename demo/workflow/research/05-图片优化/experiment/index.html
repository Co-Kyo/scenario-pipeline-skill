📋 验证检查点

  1. 懒加载请求数对比：无优化模式页面加载即发起 30 张图片请求；loading="lazy" 和 IntersectionObserver 模式下，初始请求数应 ≤ 5（仅视口内图片）。打开 DevTools → Network 面板验证。
  2. IntersectionObserver 加载时间：IO 模式下滚动到图片区域时才触发加载，首屏加载时间应明显短于无优化模式。观察性能指标面板中的「首屏加载时间」数值。
  3. srcset 尺寸选择：调整浏览器窗口宽度后，srcset 演示区应显示对应宽度的图片（400px 窗口选小图，1200px+ 窗口选大图）。拖拽窗口边缘改变宽度验证。
  4. content-visibility Layout 耗时：启用 content-visibility: auto 的 section 列表，Layout 耗时应显著低于未启用版本（通常减少 50% 以上）。点击「运行 Layout 基准测试」按钮对比。
  5. 格式降级链：在支持 AVIF 的浏览器中显示 AVIF 标签，不支持时降级到 WebP，再不支持降级到 JPEG。观察格式降级演示区当前激活的格式标签。
