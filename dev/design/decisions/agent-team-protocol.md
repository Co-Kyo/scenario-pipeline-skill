# Agent Team 决策协议

> 状态：生效
> 创建：2026-06-15

## 核心理念

复杂决策需要多角色交叉讨论，而非单角色线性规划。Agent Team 模式通过并行 spawn 多个专业 Agent，从不同视角评审方案，最终综合得出最优解。

## 触发条件

| 场景 | 示例 | 必要性 |
|------|------|--------|
| 新增入口文件 | TEST.md、PATCH.md | 必须 |
| 模块边界调整 | processes/ 拆分、core/ 重构 | 必须 |
| 跨层设计变更 | 测试策略、错误处理 | 必须 |
| 重构范围确定 | 改什么、保留什么 | 建议 |
| 架构权衡 | 多种方案对比 | 建议 |

**跳过条件**：
- Bug 修复（根因明确）
- Typo 或格式修改
- 单文件小改动

## Agent 角色定义

### 架构师（Architect）

**关注点**：整体结构、耦合风险、抽象层级

**核心问题**：
1. 这个抽象层级对吗？
2. 耦合/内聚风险是什么？
3. 有没有更简单的替代方案？

**输出**：
- 方案优点
- 主要风险
- 替代方案
- 改进建议

### 测试专家（Tester）

**关注点**：可测试性、验证流程、边界条件

**核心问题**：
1. 怎么验证这个方案有效？
2. 什么会静默失败？
3. 边界条件有哪些？

**输出**：
- 测试策略
- 验证流程
- 风险点

### 实现者（Implementer）

**关注点**：可行性、实现细节、依赖关系

**核心问题**：
1. 实现需要什么上下文？
2. 技术依赖是什么？
3. 哪里可能出错？

**输出**：
- 实现步骤
- 所需上下文
- 边界条件

## 执行流程

### Step 1: 定义问题

```markdown
## 问题
<我们要解决什么？>

## 约束
<有什么限制？>

## 目标
<成功是什么样子？>

## 提议方案
<初步想法>
```

### Step 2: 并行 Spawn Agent

```python
# 架构师
spawn(
    subagent_type="general",
    description="Architect perspective",
    prompt="""你是一个架构师 Agent，负责评审 {主题}。

## 背景
{context}

## 当前问题
{problem}

## 提议方案
{solution}

## 你的任务
从架构角度评审，提出：
1. 方案优点和风险
2. 替代方案
3. 改进建议

200字以内。"""
)

# 测试专家
spawn(
    subagent_type="general",
    description="Tester perspective",
    prompt="""你是一个测试专家 Agent，负责评审 {主题}。

## 背景
{context}

## 当前问题
{problem}

## 提议方案
{solution}

## 你的任务
从测试角度评审，提出：
1. 可测试性
2. 验证流程
3. 边界条件

200字以内。"""
)

# 实现者
spawn(
    subagent_type="general",
    description="Implementer perspective",
    prompt="""你是一个实现者 Agent，负责评审 {主题}。

## 背景
{context}

## 当前问题
{problem}

## 提议方案
{solution}

## 你的任务
从实现角度评审，提出：
1. 实现细节
2. 所需上下文
3. 风险点

200字以内。"""
)
```

### Step 3: 收集反馈

等待所有 Agent 完成，综合为表格：

```markdown
| 角色 | 核心观点 | 风险 |
|------|----------|------|
| 架构师 | ... | ... |
| 测试专家 | ... | ... |
| 实现者 | ... | ... |
```

### Step 4: 综合分析

1. **识别共识点**：多个角色都认同的观点
2. **识别分歧点**：角色间有冲突的观点
3. **权衡取舍**：根据优先级做决定

### Step 5: 决策

- 如果共识明确 → 直接决策
- 如果分歧严重 → 请示用户

## 与其他协议的关系

| 协议 | 何时使用 |
|------|----------|
| Agent Team | 架构级决策 |
| TDD | 功能开发 |
| Checkpoint | 步骤间确认 |

Agent Team 可以嵌套使用：在 TDD 流程中，如果遇到设计决策，可以启动 Agent Team 讨论。
