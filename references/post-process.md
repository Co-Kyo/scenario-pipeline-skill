# 后处理——命题深度研究

> 本文件定义后处理流程。前处理流程见 `preProcess.md`。
> 共享评估框架见 `core/scenario-matrix.md`。

---

## 一、流程定位

后处理是知识生产线的第二阶段。前处理负责"发现什么值得研究"，后处理负责"怎么研究透"。

```
前处理产出（候选池） ──→ 后处理消费（深度研究） ──→ 结构化知识资产
```

---

## 二、触发方式

```
研究：<场景描述>
```

或：

```
deep research：<场景描述>
```

**示例：**
- `研究：HTTP 缓存链路在 Nginx + Node + Webpack 构建下的完整工作机制`
- `研究：AI 聊天窗口的虚拟长列表渲染——10 万条消息，需要滚动锚定与动态高度`

**可选参数：**

| 参数 | 用法 | 说明 |
|------|------|------|
| 深度 | `研究：<场景> --depth=deep` | `shallow`（面试向摘要）/ `normal`（默认）/ `deep`（源码级分析） |
| 平台 | `研究：<场景> --platform=miniapp` | 约束范围：`web` / `miniapp` / `rn` / `all` |
| 跳过实验 | `研究：<场景> --no-experiment` | 只输出理论分析，不生成实验代码 |
| 追加 | `研究：<场景> --append` | 在已有研究目录基础上补充，不覆盖 |
| 批量 | `研究：--batch=pending` | 对候选池所有 pending 依次执行 |
| 条件筛选 | `研究：--filter="优先级=high"` | 按条件筛选后批量执行 |

## 三、研究流程

### Step 1：场景解析

从触发指令中提取：
- **核心技术域**（涉及哪些技术栈）
- **业务约束**（数据量、平台、网络环境等）
- **研究深度**（shallow / normal / deep）

**架构分词（必选）**：执行 `core/architecture-decomposition.md` 的分词方法：
- 识别**限定词**（框架/平台）和**技术栈关键词**（工程机制）
- 标注 [通用内核] 和 [特化层]，**不拆分命题**
- 确定内容权重（通用占比 ≥ 80% / 60-80% / 40-60% / < 40%）

分词结果直接决定四象限研究的覆盖范围和内容比例：
- **通用内核**是研究主体，四象限展开的核心内容
- **特化层**是框架视角，用于开篇切入和收尾落地
- 象限 I 链路解构必须覆盖所有通用工程层，不能只在框架层打转

### Step 2：四象限研究

使用 `core/scenario-matrix.md §三` 的四象限框架依次展开：

1. **链路解构** → 全链路节点梳理
2. **极致场景** → 坑点与降级策略
3. **Trade-off 雷达** → 多路线对比表
4. **MVE 实验** → 15 分钟最小可验证环境

### Step 3：结构化输出

写入 `workflow/research/<场景slug>/` 目录：

```
workflow/research/<场景slug>/
├── overview.md          # 链路解构 + 全景梳理
├── edge-cases.md        # 极致场景与降级策略
├── trade-offs.md        # 多路线对比表
├── experiment/          # 最小可验证环境
│   ├── README.md        # 实验说明与验证步骤（按下方模板）
│   └── src/             # 源代码（如适用）
└── references.md        # 参考资料链接
```

#### experiment/README.md 强制模板

```markdown
# MVE: <命题名称>

## 环境基线
- Node: v20.x
- 浏览器: Chrome v121+
- 包管理器: pnpm@8.x

## 一键启动
pnpm install && pnpm dev

## 验证检查点
1. [预期现象] → 检查方式
2. [预期现象] → 检查方式
3. [预期现象] → 检查方式

## 故障排除
- **问题A** → 解决方法
- **问题B** → 解决方法
```

> **注意**：若指令携带 `--no-experiment` 参数，跳过 experiment/ 目录生成。

### Step 4：摘要回传

研究完成后，向用户输出不超过 200 字的摘要：
- 该场景的核心链路（一句话）
- 最大的坑点（一个）
- 推荐的首选技术路线（一个，附理由）
- 实验目录路径

---

## 四、备注

- 研究产出为 Agent 基于公开知识的结构化整理，建议结合 MVE 实验建立直觉后在真实项目中验证。
- 单一技术点（如"CSS Flexbox 用法"）不在研究范围内，属于非复合工程场景命题。
